# 数据查询与现状分析

## System Prompt

你是一个智能家居适老化行业数据分析专家。你的任务是：
1. 解析用户的问题，提取关键查询条件（时间、地域、指标等）
2. 从知识库中检索相关数据
3. 用简洁明了的方式呈现数据，附上数据来源和时间
4. 如果数据不足，明确告知用户缺失什么数据

输出格式：数据表格（时间 | 指标 | 数值 | 来源）+ 简要解读（2-3句话）

---

## 核心模块一：RAG 检索增强生成

采用"结构化数据库 + 向量知识库"的二段式 RAG 架构：

### 第一层：结构化检索（PostgreSQL）
- 存储结构化数据：人口统计表、市场规模表、企业信息表、政策文档表等
- 支持 SQL 精确查询，适合"查2024年市场规模"这类精确问题

### 第二层：语义检索（FAISS / ChromaDB）
- 存储非结构化知识的向量嵌入：政策全文、研究报告摘要、学术论文片段等
- 支持语义相似度检索，适合"养老政策对智能家居有什么影响"这类语义问题
- 使用 `shibing624/text2vec-base-chinese` 作为嵌入模型

### 混合查询流程
```
用户查询 → 提取关键词 → PostgreSQL精确检索 + ChromaDB语义检索
                            → 结果融合 → 去重排序 → 格式化输出
```

---

## 核心模块二：ETL 数据处理

数据经过 ETL（Extract-Transform-Load）流程处理后入库：

| 阶段 | 操作 | 说明 |
|:---|:-----|:-----|
| **Extract** | 数据采集 | 从各数据源提取原始数据：API调用、网页爬虫、文件下载 |
| **Transform** | 数据清洗 | 统一时间戳格式、处理缺失值、去重、格式转换、标准化 |
| **Load** | 数据加载 | 结构化数据 → PostgreSQL；文本切片向量化 → FAISS/ChromaDB |
| **QC** | 质量控制 | 政府数据与学术数据交叉验证，确保数据一致性 |

---

## 核心模块三：数据三角互证

通过三大类数据源的交叉验证，确保数据质量和分析结论的可靠性：

```
                    ┌─────────────────────────┐
                    │  政府公开数据            │
                    │  (国家统计局/民政部/工信部)│
                    └──────────┬──────────────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
          ▼                    ▼                    ▼
  ┌───────────────┐   ┌───────────────┐   ┌────────────────┐
  │ 学术研究数据   │◄──┤  数据交叉验证  ├──►│ 社会调查数据    │
  │ (CASAS/Kaggle) │   │  (一致性检验)  │   │ (CFPS/CLASS)   │
  └───────────────┘   └───────────────┘   └────────────────┘
```

- **政府数据** → 宏观指标（人口结构、产业规模、政策投入）
- **学术数据** → 研究数据（技术趋势、用户行为实验）
- **社会调查** → 微观数据（家庭追踪、消费偏好、养老需求）

三者交叉验证示例：用国家统计局的人口数据验证 CFPS 样本代表性，用学术研究报告佐证市场趋势判断。

## 输出规范

1. 结构化数据表格优先
2. 标注数据获取时间和来源
3. 缺失数据明确标注
4. 附简要趋势解读
5. 多源数据冲突时，标注数据三角互证结果
