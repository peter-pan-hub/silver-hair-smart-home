# 趋势分析与预测

## System Prompt

你是行业趋势预测专家，擅长时间序列分析。你的任务是：
1. 确认用户需要预测的指标和时间范围
2. 从数据库中提取历史数据
3. 运行多模型集成预测（ARIMA + LSTM + Prophet）
4. 输出预测结果：
   - 预测值（含95%置信区间）
   - 各模型权重与一致性说明
   - 影响预测的关键因素分析
5. 标注预测的局限性（数据长度、外部冲击等）

---

## 核心模块一：RAG 趋势检索增强生成

针对趋势预测的数据检索，采用二段式架构：

### 第一层：结构化检索（PostgreSQL）
- 存储时序指标表：市场规模、老龄化率、渗透率、政策投入等历年数据
- 支持按指标名称、地域、时间范围精确筛选
- 适合"2020-2024年中国智能家居市场规模"这类精确查询

### 第二层：语义检索（FAISS / ChromaDB）
- 存储趋势分析报告、行业研报片段的向量嵌入
- 支持语义相似度检索，适合"哪些因素驱动了智能家居适老化市场增长"这类语义问题
- 使用 `shibing624/text2vec-base-chinese` 作为嵌入模型

### 混合检索流程
```
预测请求 → 提取指标+地域+时间范围
    → PostgreSQL精确检索（时序指标历史数据）
    + ChromaDB语义检索（行业分析报告/影响因素描述）
    → 数据融合 → 输入多模型集成预测
```

---

## 核心模块二：ETL 时序数据处理

预测所需的历史数据经过 ETL 流程处理后入库：

| 阶段 | 操作 | 说明 |
|:---|:-----|:-----|
| **Extract** | 数据采集 | 从统计局数据库、行业报告、企业财报、公开数据集中提取时序数据 |
| **Transform** | 数据清洗 | 统一时间粒度（年/季/月）、处理缺失值（插值法）、剔除异常值、对数平滑 |
| **Load** | 数据加载 | 清洗后的时序数据 → PostgreSQL（时序表）；分析报告片段向量化 → FAISS/ChromaDB |
| **QC** | 质量控制 | 多源数据交叉核对（如统计局 vs 行业协会口径差异），确保时序一致性 |

## 核心模块三：预测数据三角互证

通过三大类数据源的交叉验证，确保预测基础的可靠性：

```
                  ┌──────────────────────────┐
                  │  政府统计数据              │
                  │  (统计局/民政部年报)       │
                  └────────────┬─────────────┘
                              │
         ┌────────────────────┼────────────────────┐
         │                    │                    │
         ▼                    ▼                    ▼
 ┌───────────────┐   ┌───────────────┐   ┌────────────────┐
 │ 行业研究报告   │◄──┤  数据交叉验证  ├──►│ 企业实际数据    │
 │ (艾瑞/中商产业)│   │  (趋势一致性)  │   │ (上市公司财报)  │
 └───────────────┘   └───────────────┘   └────────────────┘
```

- **政府统计数据** → 宏观时序基准（GDP、老龄化率、养老产业投入）
- **行业研究报告** → 市场估算数据（市场规模、渗透率、增长率）
- **企业实际数据** → 微观验证（头部企业营收变化、产品出货量）

**三角互证示例**：用国家统计局的老龄化率预测 → 对比艾瑞咨询的市场规模报告 → 验证于头部企业（如安居宝、小米）智能家居适老化业务营收增长趋势。

## 核心模块四：多模型集成预测

### 模型权重分配策略

| 模型 | 基础权重 | 适用场景 | 权重上调条件 |
|:----|:-------:|:--------|:-----------|
| ARIMA | 0.3 | 稳定线性增长趋势 | 数据呈明显线性趋势时 +0.1 |
| LSTM | 0.4 | 非线性复杂模式 | 历史波动较大、存在拐点时 +0.1 |
| Prophet | 0.3 | 周期性/节假日效应 | 季节性明显（如Q4旺季）时 +0.1 |

**权重调整规则**：
- 三个模型的基础权重和为 1.0（0.3 + 0.4 + 0.3）
- 某个模型上调权重时，其余两个模型等比下调，保持总权重 = 1.0
- 输出时标注各模型的实际权重值和调整理由

### 预测工作流节点

| 节点 | 功能 | 技术实现 | 预期耗时 |
|:----|:----|:--------|:--------|
| 参数解析 | 提取预测目标/时间范围 | 正则+LLM | <0.5秒 |
| 数据准备 | 从PostgreSQL提取历史数据 | psycopg2+pandas | <1秒 |
| ARIMA预测 | 线性趋势预测 | statsmodels.ARIMA | <2秒 |
| LSTM预测 | 非线性模式捕捉 | TensorFlow/Keras | <5秒 |
| Prophet预测 | 周期性/节假日效应 | fbprophet | <3秒 |
| 集成加权 | 多模型结果融合 | 加权平均/Stacking | <0.5秒 |
| 结果输出 | 预测值+置信区间+图 | matplotlib+LLM | <2秒 |

### 集成预测流程

```
用户输入（预测请求）
  │
  ▼
① 参数解析（预测指标 + 时间范围）
  │
  ▼
② RAG混合检索 + ETL时序数据准备（PostgreSQL → pandas DataFrame）
  │
  ├──→ ③ ARIMA 预测（线性趋势）
  ├──→ ③ LSTM 预测（非线性模式）
  └──→ ③ Prophet 预测（周期性效应）
  │
  ▼
④ 集成加权计算（动态权重分配）
  │
  ▼
⑤ 结果输出（预测值 + 95%置信区间 + 权重说明 + 三角互证验证结果）
```

## 预测模型

| 模型 | 用途 | 优势 |
|------|------|------|
| ARIMA | 线性趋势预测 | 可解释性强 |

|------|--------|

## 输出规范

1. 标注预测使用的历史数据来源和时间范围

3. 报告各模型权重及调整理由
