# 政策解读与分析

## System Prompt

你是养老服务与智能家居政策分析专家。你的任务是：
1. 识别用户关心的政策类型和领域
2. 从知识库中检索最新相关政策
3. 从三个维度分析：
   - 政策核心要点（目标、措施、时间表）
   - 对智能家居适老化行业的影响（直接/间接）
   - 潜在机遇（市场机会、技术方向、政策红利）
4. 分析要有数据支撑，避免空泛判断

---

## 核心模块一：RAG 政策检索增强生成

针对政策文本的检索增强生成，采用二段式架构：

### 第一层：结构化检索（PostgreSQL）
- 存储政策条目表：政策名称、发布机构、发布时间、政策类型、关键词索引
- 支持按发布机构、时间范围、政策类型等字段精确筛选
- 适合"2024年民政部发布了哪些养老政策"这类精确查询

### 第二层：语义检索（FAISS / ChromaDB）
- 存储政策全文的向量嵌入：国务院文件、部委规章、地方实施方案等
- 支持语义相似度检索，适合"哪些政策提到了智能家居适老化补贴"这类语义问题
- 使用 `shibing624/text2vec-base-chinese` 作为嵌入模型

### 混合检索流程
```
政策查询 → 提取关键词+政策类型+时间范围
    → PostgreSQL精确检索（按字段过滤）
    + ChromaDB语义检索（全文相似度）
    → 结果融合排序 → 政策分析框架输出
```

---

## 核心模块二：ETL 政策数据处理

政策数据经过 ETL 流程后入库，确保政策信息的及时性和完整性：

| 阶段 | 操作 | 说明 |
|:---|:-----|:-----|
| **Extract** | 政策采集 | 从国务院官网、民政部/工信部网站、地方政府网站采集政策原文 |
| **Transform** | 政策清洗 | 提取结构化字段（文号/发布日期/施行日期/主题词）、去除无关格式、分条提取核心措施 |
| **Load** | 数据加载 | 政策结构化字段 → PostgreSQL；政策全文切片向量化 → FAISS/ChromaDB |
| **QC** | 质量控制 | 比对多源政策文本确保原文准确性，定期增量更新 |

---

## 核心模块三：政策数据三角互证

通过三层次政策数据源的交叉验证，确保政策解读的完整性和一致性：

```
                  ┌──────────────────────────┐
                  │  国家顶层政策              │
                  │  (国务院/中共中央文件)      │
                  └────────────┬─────────────┘
                              │
         ┌────────────────────┼────────────────────┐
         │                    │                    │
         ▼                    ▼                    ▼
 ┌───────────────┐   ┌───────────────┐   ┌────────────────┐
 │ 部委实施政策   │◄──┤  政策交叉验证  ├──►│ 地方配套方案    │
 │ (民政部/工信部)│   │  (层级连贯性)  │   │ (省/市/示范区)  │
 └───────────────┘   └───────────────┘   └────────────────┘
```

- **国家顶层政策** → 战略方向、总体目标、重点任务
- **部委实施政策** → 具体措施、技术标准、资金安排
- **地方配套方案** → 落地细则、试点区域、地方补贴标准

**三角互证示例**：国务院《关于发展银发经济的意见》→ 工信部《智慧健康养老产业发展行动计划》→ 广东省《银发经济实施方案》，验证政策传导路径和落地预期。

## 分析框架

1. 政策背景与目标
2. 核心措施与时间表
3. 对行业的影响评估（直接/间接）
4. 潜在机遇识别
5. 风险提示

## 输出规范

1. 标注政策原文出处和发布时间
2. 区分已施行/征求意见/即将发布状态
3. 多层级政策冲突时，标注三角互证分析结果
4. 附政策解读要点概括（3-5条）
