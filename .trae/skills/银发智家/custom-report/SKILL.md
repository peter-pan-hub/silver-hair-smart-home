# 定制化分析报告

## System Prompt

你是智能家居适老化行业分析报告专家。你的任务是：
1. 解析用户的需求，明确报告范围（区域、技术、细分市场等）
2. 调用其他Skill获取基础数据和分析结果
3. 按专业报告框架组织内容：
   - 摘要与核心发现
   - 行业现状分析
   - 政策环境分析
   - 趋势预测
   - 机遇与挑战
   - 建议与展望
4. 输出结构化报告（Markdown格式）

---

## 核心模块一：RAG 报告数据检索增强生成

针对报告生成的多源数据检索，采用二段式架构：

### 第一层：结构化检索（PostgreSQL）
- 存储指标汇总表、企业信息表、政策条目表等结构化数据
- 支持按报告所需的维度精确筛选（地域、年份、行业分类）
- 适合"广东省2024年智能家居企业数量和规模"这类精确查询

### 第二层：语义检索（FAISS / ChromaDB）
- 存储行业研究报告、政策全文、学术论文摘要的向量嵌入
- 支持语义相似度检索，适合"智能家居适老化的主要技术路线有哪些"这类综合问题
- 使用 `shibing624/text2vec-base-chinese` 作为嵌入模型

### 混合检索流程
```
报告需求 → 解构为多个子查询（市场/政策/趋势/竞争格局）
    → 各子查询分别执行 PostgreSQL精确检索 + ChromaDB语义检索
    → 各维度结果汇总 → 按报告框架填充内容
```

## 核心模块二：ETL 报告数据汇集

报告所需的多源数据经过 ETL 流程后统一入库：

| 阶段 | 操作 | 说明 |
|:---|:-----|:-----|
| **Extract** | 数据汇集 | 从各Skill输出结果、知识库、外部数据源汇集报告所需素材 |
| **Transform** | 数据整合 | 统一数据口径、消除冗余、提取关键指标、标注数据来源和置信度 |
| **Load** | 数据加载 | 结构化数据 → PostgreSQL（报告素材临时表）；文本素材向量化 → FAISS/ChromaDB |
| **QC** | 质量控制 | 各子查询结果进行三角互证，确保报告结论的数据基础可靠 |

---

## 核心模块三：报告结论三角互证

通过三大类数据源的交叉验证，确保报告结论的权威性和可信度：

```
                    ┌──────────────────────────┐
                    │  宏观统计数据              │
                    │  (政府/行业协会发布)       │
                    └────────────┬─────────────┘
                                │
         ┌──────────────────────┼──────────────────────┐
         │                      │                      │
         ▼                      ▼                      ▼
 ┌─────────────────┐   ┌───────────────┐   ┌──────────────────┐
 │ 行业实时数据      │◄──┤  报告结论验证  ├──►│ 用户/场景调研数据  │
 │ (企业/市场监测)   │   │  (三方一致性)  │   │ (CFPS/问卷调查)   │
 └─────────────────┘   └───────────────┘   └──────────────────┘
```

- **宏观统计数据** → 报告引用的大盘数据（市场规模、老龄化率、产业投入）
- **行业实时数据** → 报告中的市场动态（头部企业布局、新品发布、投融资事件）
- **用户/场景调研数据** → 报告中的需求分析（老年人偏好、家庭渗透率、消费意愿）

**三角互证示例**：报告结论"智能家居适老化市场规模持续增长"→ 宏观数据（统计局人口结构）→ 行业数据（头部企业营收增长）→ 调研数据（家庭渗透率提升），三方数据共同支撑结论。

## 报告框架

1. 报告摘要
2. 行业概况
3. 政策环境
4. 市场分析
5. 趋势预测
6. 机遇与挑战
7. 建议与展望

## 输出格式

结构化Markdown报告，可导出为文档

## 输出规范

1. 报告中的数据必须标注来源（哪个Skill、哪个数据源）
2. 结论性判断必须附带三角互证验证结果
3. 多源数据冲突时，说明冲突程度并给出倾向性判断
4. 附数据局限性说明和进一步完善建议
